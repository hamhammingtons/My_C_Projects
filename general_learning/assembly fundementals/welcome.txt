How does the CPU communicate with the other components of the circuit? 

There are 3 things called 'busses'. they handle the communcation 

Control bus;
Control bus is made just for synchronization within the devices that are attached to the bus.
Example: where are we writing, reading, interacting at the given time 

Adress bus; 
Adress bus is made for holding the 'adress' of instructions and 'data' that ARE BEING TRANSFERED between CPU and memory
or any other devices.
Think of it like a void pointer: it just points to 'different places' where things ARE GOING TO BE MOVED TO 
Example: tells us where were working at in the hardware scheme

Data bus;
Data bus handles the 'transfer of instructions' between the data of: cpu, memory, i/o devices
But usually i/o devices exist on a seperate bus called the i/o bus

CPU components:
    High frequency clock
    Control unit
    ALU - arichmetic logic unit 
    [Memory] Registers - storage location {usually refered as Memory Registers}

ALU is the part of the cpu that carries LOGIC and ARICHMETIC:
    Does operations like add, subtract, and, or, not 
    'Special part of the cpu that does the calculations'

Memory Registers are the type of 'computer memory' that are very close to the cpu:
    Fastest possible memory for us to access 
    'We will always try to store things in the memory registers before we store them in external things [RAM and etc]'
