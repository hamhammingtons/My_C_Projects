How does the CPU communicate with the other components of the circuit? 

There are 3 things called 'busses'. they handle the communcation 

Control bus;
Control bus is made just for synchronization within the devices that are attached to the bus.
Example: where are we writing, reading, interacting at the given time 

Adress bus; 
Adress bus is made for holding the 'adress' of instructions and 'data' that ARE BEING TRANSFERED between CPU and memory
or any other devices.
Think of it like a void pointer: it just points to 'different places' where things ARE GOING TO BE MOVED TO 
Example: tells us where were working at in the hardware scheme

Data bus;
Data bus handles the 'transfer of instructions' between the data of: cpu, memory, i/o devices
But usually i/o devices exist on a seperate bus called the i/o bus

        short but simple:

        - Frequency (f): How often a cycle repeats over time (f = 1/T). Measured in Hertz (Hz).

        - Amplitude: The strength or voltage level.
            - Peak Amplitude: Distance from the center (X-axis) to the top (Crest).
            - Peak-to-Peak: Distance from the highest point (Crest) to the lowest (Trough).
        
        - Period (T): The horizontal TIME of 1 complete cycle. Measured IN SECONDS. <---   
            'how much it takes for 1 cycle to pass'
        - wavelength(lambda): The spatial length of 1 wave cycle. Measured IN METERS <----

        - 'Angular' Frequency (w): The "rotational speed" of the wave in radians per second (w = 2* 'pi' *f).

        - Wavenumber (k): Represents the number of waves over a distance (k = 2*pi/lambda).

        - Phase (phi): Determines where the wave starts at t=0. Shifting the wave left or right changes its phase, 
            which is vital for syncing different hardware parts.

        STARTING POINTS FOR USING ANY FORMULA:
        1. Amplitude (A)

        3. Frequency (f)
        4. Phase

        \0 of all simple to math:
        
        f = how often a cycle repeats over time [1/T] (Hertz)
            linear formula: f = 1/T
            Angular formula: ω = 2πf (rotational speed in radians/sec)
            \
            Angular Callname: Angular frequency

        A = Amplitude: Distance from the center (X-axis) to the top (Crest)
            linear formula: A (peak height)
            Angular formula: A * sin(...) (multiplies inside sine function)
            \
            Angular Callname: Angular Amplitude

        T = period = Time of 1 complete cycle [seconds]
            linear formula: T = 1/f
            Angular formula: T = 2π/ω (time per full rotation)
            \
            Angular Callname = Time component

        lambda = wavelength - length of 1 wave cycle [meters]
            linear formula: λ = u/f (distance per cycle)
            Angular formula: k = 2π/λ (radians per meter spatial density)
            \
            Angular Callname = Wavenumber

        u = wave speed - The distance the wave travels per unit of time
            linear formula: u = f * λ
            Angular formula: u = ω/k (radians cancel out)

        phi = phase = Determines where the wave starts at t = 0
            linear formula: φ (start offset in cycles)
            Angular formula: sin(k*x - ω*t + φ) (full phase argument)

        Why do we even need Linear and Angular?
        Linear = Counting full laps (for humans).
        Angular = Calculating the exact angle/position on the lap (for calculations). [usually Radians].

        So linear is all about simplicity. Angular is about the numbers. 
        Example being:
        The CPU does 10 cycles per second.
            f = 10 Hertz
        What total distance has your signal passed? (specific for calculations)
            angular'f'[w] = 2 * pi * f = (2*10) * pi = 20pi = 62.8 radians
        
        Then, if youre being asked for example 'what resistance on the data bus will you have in exactly 0.05 seconds' then 
            voltage = A * sin(w * t) = A * sin(62.8 * 0.05)
